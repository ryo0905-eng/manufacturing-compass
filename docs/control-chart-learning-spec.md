# 管理図インタラクティブ学習ツール仕様

最終更新日: 2026-07-19

## 目的

管理限界を計算するだけでなく、時系列の並びから偶然原因と異常原因を区別し、次に何を確認するかを学ぶ。Excel、JMP、Minitabなどの代替ではなく、それらの出力を判断する前提を身につける。

## 最初の対象

- 1時点につき測定値1個の I 管理図を入口にする
- 24点の固定学習データをブラウザ内で生成する
- 最初の安定期間から中心線と管理限界を推定し、その後の変化を監視する
- 入力値や操作状態を保存・外部送信しない

Xbar-R、Xbar-S、p、np、c、u管理図は同時実装しない。I管理図で学習体験を検証してから追加する。

## 学習フロー

### 1. 正常な散らばりを見る

点が管理限界内にあり、中心線の上下へランダムに並ぶ状態から始める。「限界内だから絶対正常」ではなく、並び方も見ることを先に示す。

### 2. 異常を混ぜる

ユーザーが次のシナリオを切り替える。

- 正常工程
- 平均シフト: 途中から中心が移動する
- 単発外れ: 1点だけ大きく外れる
- 傾向: 連続して増加または減少する
- 周期変動: 一定周期で上下する

数値、管理図、検出ルール、解釈を同時に更新する。

### 3. 検出ルールを理解する

初期版では、説明しやすい次のルールに限定する。

- 1点が3σ管理限界の外側
- 8点連続で中心線の同じ側
- 6点連続で増加または減少

ルール番号を暗記させず、どの点の並びが検出されたかを管理図上で強調する。ルールの採用基準は企業・規格・ソフト設定で異なることを明示する。

### 4. 次の確認へつなぐ

検出後は自動的に原因を断定しない。シナリオに応じて、設備条件、材料ロット、測定器、作業者、保全、環境、実施記録などの確認候補を示す。

## 必須の誤解防止

### 管理限界と規格限界

- 管理限界: 工程データから求める、工程変動を監視する境界
- 規格限界: 顧客・設計・社内基準で決まる、製品要求の境界
- 管理状態でも規格を満たすとは限らない
- 規格内でも非ランダムな変化があれば管理状態とは限らない

画面では両者を同じ線種・色で表示しない。初期の管理図では規格線を非表示にし、比較操作をした時だけ追加する。

## UI構成

```text
シナリオ選択
  → 時系列管理図
  → 検出された並びを点と帯で強調
  → 「何が見えたか」
  → 「次に何を確認するか」
```

- 正常状態を初期値にする
- 状態色だけで異常を伝えず、点記号と文章を併用する
- グラフはSeabornに近い薄いグレー背景、青い測定値、意味のある限界線だけを使う
- スマートフォンではグラフを縮小しすぎず、必要なら横スクロールする
- 詳細理論は折りたたみ、操作と判断を初期画面で完結させる

## 計算方針

- I管理図の工程σは移動範囲から推定する
- `MRbar = 隣接2点の移動範囲の平均`
- `sigma = MRbar / 1.128`
- `UCL / LCL = center ± 3 sigma`
- 学習用データでは限界算出期間と監視期間を画面上で区別する
- 計算関数はUIから分離し、固定データに対する期待値を検証可能にする

## 計測

- ツール開始
- シナリオ選択
- 検出ルールの詳細表示
- 管理限界と規格限界の比較操作
- Cp・CpkまたはDoEへの関連遷移

測定値そのものはAnalyticsへ送信しない。

## 初期版の完了条件

- 5シナリオが切り替わり、管理図と解釈が即時更新される
- 3種類の非ランダムルールが該当点を強調する
- 管理限界と規格限界の違いを操作で確認できる
- 検出後に原因を断定せず、次の確認候補を示す
- PCとスマートフォンで主要操作ができる
- `/tools`とトップページからアクセスできる

# Design System

最終更新日: 2026-07-21

## デザイン原則

Manufacturing Compass は、製造技術者が情報を速く正確に読める、静かで密度のあるUIを目指します。管理画面や技術ドキュメントのような整然さを持たせつつ、長い日本語を無理なく読める余白と文字サイズを保ちます。

- モバイルファースト
- 読みやすい余白と明確な情報階層
- エンジニア向けの技術ドキュメントに近い簡潔さ
- 派手な装飾より内容を優先
- 広告サイトらしい煽りを避ける
- 判定される印象ではなく、前向きに整理できる印象
- 色やカードではなく、整列・余白・文字階層で情報を分ける
- 同じページ内で複数のデザイン表現を混在させない

## Quest 表現

Career Compass に限り、探索を助ける軽い Quest 表現を使えます。例外として、キャリア準備ページでは小さな行動提案に `Today Quest` だけを使えます。これはデザイン全体の必須テーマではありません。

使ってよい表現:

- Quest
- Route
- Stage
- Skill
- Today Quest
- Power Score（必ず「転職材料の整理度」を併記する）

避ける表現:

- 勇者、ラスボス、討伐
- レベルが低い
- 勝ち・負け
- 人の価値を示す Career Power
- 年収を報酬として扱う Reward Range

スコアは「転職材料の整理度」、年収は「職種別の参考年収レンジ」と説明します。

## 色と装飾

- 背景: `#ffffff`
- 補助背景: `#f7f7f8` または `#fafafa`
- 本文: `#1d1d1f`
- 補助本文: `#60666d`
- 通常罫線: `#e5e7eb`
- 強い罫線: `#c9cdd2`
- 操作色: `#1769aa`
- 図表・選択色: `#4c72b0`
- 選択背景: `#eef3fa`
- 主CTA: `#1d1d1f`

操作色はリンク、矢印、フォーカスなど「操作できる場所」に使います。図表・選択色は、選択中の要素や同じ系列を追うために使い、通常の見出し、数値、カード枠を青緑で装飾しません。成功・注意・エラーなど意味を持つ状態色は、その状態を伝える箇所だけで使います。

グラデーション、発光、強い影、装飾目的のグリッドは通常ページでは使いません。Career Compass の結果補足や技術図解で関係を示す場合だけ、必要最小限に限定します。色だけで状態を伝えません。

## タイポグラフィとレイアウト

- 基本フォントはOSのシステムサンセリフを使う
- 本文は16px、補足は12〜13px、ナビとボタンは13〜14pxを基準にする
- 通常ページの `h1` は32〜36px、セクションの `h2` は24〜28pxを基準にする
- 見出しは600、本文は400、ラベルは500〜600を基本とし、太字を重ねすぎない
- 外側のコンテンツ幅は1080px、読み物の本文幅は680pxを基準にする
- セクション間隔はPCで64px、スマホで48pxを基準にする
- 本文はスマホで無理なく読める行長にする
- 見出しでページの結論と役割を理解できるようにする
- 数字は単位・対象時点・注記と一緒に表示する
- 表は比較関係がある時だけ使い、スマホでは横はみ出しを避ける
- カードを使いすぎず、本文の流れが自然な時は文章を使う
- 同じ説明を本文、カード、注記で繰り返さない

## ページ種別ごとの密度

ページの役割に合わせ、次の3種類を基準にします。同じサイトであることが伝わるように、色、フォント、罫線、CTAの原則は共通にします。

### インタラクティブ学習・探索ページ

Cp・Cpk、DoE、管理図、Gage R&R、業界地図のように、操作しながら関係を理解する画面の標準です。

- 外側の最大幅は1180px、ページ上下は `12px 0 72px` を基準にする
- ページ内の主要な間隔は12pxを基準にし、操作領域と説明領域を近くに置く
- `h1` は26〜28px、主要パネルの `h2` は18〜22px、本文は13〜14pxを基準にする
- 英字のアイブロウ、列見出し、補足ラベルは9〜11pxを基準にする
- 操作パネルは白または `#f7f7f8`、1pxの通常罫線、8px以下の角丸、影なしを基本にする
- 選択中の要素は図表・選択色と選択背景で示し、文字やアイコンも併用する
- 操作結果、図、次に読む説明を同じ画面内で対応付ける
- デスクトップでは全体像、モバイルでは操作順と読み順を優先する

現在の参照実装は `.doe-page`、`.control-chart-page`、`.grr-page`、`.cpk-page--analysis`、`.industry-map-page` です。新しい画面は特定ページのCSSを複製せず、この共通ルールを起点にします。

### 一覧・入口ページ

企業一覧、ツール一覧、ガイド一覧など、次の行き先を選ぶ画面の標準です。

- 外側の最大幅は1080pxを基準にする
- `h1` は32〜38px、`h2` は24〜28px、本文は14〜16pxを基準にする
- 入口同士の違いは、装飾色ではなく見出し、短い説明、メタ情報で示す
- 子ページが学習ツールの場合、入口ページにも同じ中立色と文字階層を使う

### 読み物・参照ページ

ガイド、会社情報、法務・運営情報など、文章を読む画面の標準です。

- 外側の最大幅は960px、本文の読み幅は680〜740pxを基準にする
- `h1` は32〜36px、`h2` は24〜28px、本文は16pxを基準にする
- 見出し、本文、出典、更新日を余白と文字階層で区別する
- 企業詳細でも、企業固有のブランドカラーをページ全体の装飾へ使わない

## 意味のある例外

- Career Compass の質問・結果画面は、探索を助ける範囲で Quest 表現と暗色面を使える
- Today Quest は、Career Compass またはキャリア準備ページの小さな行動提案に限る
- 技術図解やグラフでは、系列、工程、状態の違いを示す色を使える。凡例または本文説明を付け、装飾だけの色にしない
- 広告・注意・成功・エラーの色は、その意味がある領域だけに限定する
- 例外の表現をページ全体や他の通常ページへ広げない。例外領域の外は共通のタイポグラフィとアクセシビリティに従う

## 実装ルール

- 共通トークンの正本より後ろに、ページ専用の `:root` を追加しない
- ページ固有の色を直接指定する場合は、図表の系列や状態など意味をコメントまたはデータ名で説明できるようにする
- グラデーション、強い影、装飾グリッドを、新しいヒーローやカードの見栄えだけを目的に追加しない
- 過去ページのCSSをそのままコピーせず、上記のページ種別と共通トークンへ合わせる
- CSSの詳細度で古い見た目を上書きし続けず、移行時に不要になった旧スタイルを対象範囲内で整理する
- PCとスマホで、見出しサイズ、読み順、横スクロール、タップ領域、フォーカス表示を確認する

実装後は、次を短く確認します。

- 見出しや数値に装飾目的の青緑を使っていないか
- 文字サイズと情報密度が同種のページと揃っているか
- 色が操作、選択、系列、状態のいずれかを説明しているか
- 影、グラデーション、大きな角丸がなくても階層を理解できるか
- 主要CTAが1セクションに一つか
- モバイルで最初に理解すべき内容と操作が見えるか

## 罫線、カード、余白

- セクションの区切りは原則として余白で示す
- 罫線は表、比較、フォーム、一覧の関係を理解するために必要な場合だけ使う
- 見出しの下へ習慣的に横線を引かない
- 一覧は、罫線よりも薄い背景と小さな間隔で行を分けることを優先する
- カードは白背景、1pxの通常罫線、4〜8pxの角丸を基本にする
- カードへ強い影を付けない
- 同じ形のカードを連続させすぎず、リストや本文も使う

## ボタンとリンク

- 主CTAは黒背景・白文字の小さなボタンを基本にする
- 補助ボタンは白背景・通常罫線にする
- テキストリンクと矢印に操作色を使う
- 同じリンク先を通常ナビと隣接CTAで重複させない
- 1セクション内で視覚的に強いCTAは一つにする

## Career Compass

- 1問ずつ進み、進捗と戻る操作を明確にする
- 選択肢はタップ領域を十分に確保する
- 任意入力は任意であることを表示する
- 結果は合否ではなく、経験の翻訳と準備として見せる
- 参考結果と公開情報を区別する
- 主結果を先に、詳しいスコア根拠や相談準備は展開可能な補足にする
- Today Quest は小さく実行できる内容にする

## コンテンツページ

- 技術記事は工程・構造・比較など、関係を理解しやすくする図解を使う
- 実体験記事は本人の言葉と時系列を優先し、カードへ分解しすぎない
- 企業ページは概要、職種、情報源、更新日、キャリア準備を見分けやすくする
- ランキングは基準時点、通貨、算出方法を表の近くに示す

記事固有のビジュアル判断は [`article-visual-guidelines.md`](./article-visual-guidelines.md) を参照してください。

## CTA

- 1セクションの主CTAを一つにする
- 押した後に何が起きるか分かる文言にする
- 相談CTAは相談論点を示す
- 広告表記は読み飛ばされるほど小さくしない
- 公式リンクと広告リンクの見た目・説明を区別する

## アクセシビリティ

- キーボード操作とフォーカス表示を保つ
- 見出し階層とランドマークを正しく使う
- 図解に代替テキストまたは本文説明を付ける
- 折りたたみの開閉状態を支援技術で理解できるようにする
- タップ領域、コントラスト、読み上げ順をモバイルで確認する

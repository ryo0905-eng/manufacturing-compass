# Design System

最終更新日: 2026-07-19

## デザイン原則

Manufacturing Compass は、製造技術者が情報を速く正確に読める、静かで密度のあるUIを目指します。管理画面や技術ドキュメントのような整然さを持たせつつ、長い日本語を無理なく読める余白と文字サイズを保ちます。

- モバイルファースト
- 読みやすい余白と明確な情報階層
- エンジニア向けの技術ドキュメントに近い簡潔さ
- 派手な装飾より内容を優先
- 広告サイトらしい煽りを避ける
- 判定される印象ではなく、前向きに整理できる印象
- 色やカードではなく、整列・余白・文字階層で情報を分ける
- 同じページ内で複数のデザイン表現を混在させない

## Quest 表現

Career Compass に限り、探索を助ける軽い Quest 表現を使えます。これはデザイン全体の必須テーマではありません。

使ってよい表現:

- Quest
- Route
- Stage
- Skill
- Today Quest
- Power Score（必ず「転職材料の整理度」を併記する）

避ける表現:

- 勇者、ラスボス、討伐
- レベルが低い
- 勝ち・負け
- 人の価値を示す Career Power
- 年収を報酬として扱う Reward Range

スコアは「転職材料の整理度」、年収は「職種別の参考年収レンジ」と説明します。

## 色と装飾

- 背景: `#ffffff`
- 補助背景: `#f7f7f8` または `#fafafa`
- 本文: `#1d1d1f`
- 補助本文: `#60666d`
- 通常罫線: `#e5e7eb`
- 強い罫線: `#c9cdd2`
- 操作色: `#1769aa`
- 主CTA: `#1d1d1f`

青はリンク、矢印、選択、フォーカスなど「操作できる場所」に限定します。通常の見出し、数値、カード枠を青緑で装飾しません。成功・注意・エラーなど意味を持つ状態色は、その状態を伝える箇所だけで使います。

グラデーション、発光、強い影、装飾目的のグリッドは通常ページでは使いません。Career Compass の結果補足や技術図解で関係を示す場合だけ、必要最小限に限定します。色だけで状態を伝えません。

## タイポグラフィとレイアウト

- 基本フォントはOSのシステムサンセリフを使う
- 本文は16px、補足は12〜13px、ナビとボタンは13〜14pxを基準にする
- 通常ページの `h1` は32〜36px、セクションの `h2` は24〜28pxを基準にする
- 見出しは600、本文は400、ラベルは500〜600を基本とし、太字を重ねすぎない
- 外側のコンテンツ幅は1080px、読み物の本文幅は680pxを基準にする
- セクション間隔はPCで64px、スマホで48pxを基準にする
- 本文はスマホで無理なく読める行長にする
- 見出しでページの結論と役割を理解できるようにする
- 数字は単位・対象時点・注記と一緒に表示する
- 表は比較関係がある時だけ使い、スマホでは横はみ出しを避ける
- カードを使いすぎず、本文の流れが自然な時は文章を使う
- 同じ説明を本文、カード、注記で繰り返さない

## 罫線、カード、余白

- セクションの区切りは原則として余白で示す
- 罫線は表、比較、フォーム、一覧の関係を理解するために必要な場合だけ使う
- 見出しの下へ習慣的に横線を引かない
- 一覧は、罫線よりも薄い背景と小さな間隔で行を分けることを優先する
- カードは白背景、1pxの通常罫線、4〜8pxの角丸を基本にする
- カードへ強い影を付けない
- 同じ形のカードを連続させすぎず、リストや本文も使う

## ボタンとリンク

- 主CTAは黒背景・白文字の小さなボタンを基本にする
- 補助ボタンは白背景・通常罫線にする
- テキストリンクと矢印に操作色を使う
- 同じリンク先を通常ナビと隣接CTAで重複させない
- 1セクション内で視覚的に強いCTAは一つにする

## Career Compass

- 1問ずつ進み、進捗と戻る操作を明確にする
- 選択肢はタップ領域を十分に確保する
- 任意入力は任意であることを表示する
- 結果は合否ではなく、経験の翻訳と準備として見せる
- 参考結果と公開情報を区別する
- 主結果を先に、詳しいスコア根拠や相談準備は展開可能な補足にする
- Today Quest は小さく実行できる内容にする

## コンテンツページ

- 技術記事は工程・構造・比較など、関係を理解しやすくする図解を使う
- 実体験記事は本人の言葉と時系列を優先し、カードへ分解しすぎない
- 企業ページは概要、職種、情報源、更新日、キャリア準備を見分けやすくする
- ランキングは基準時点、通貨、算出方法を表の近くに示す

記事固有のビジュアル判断は [`article-visual-guidelines.md`](./article-visual-guidelines.md) を参照してください。

## CTA

- 1セクションの主CTAを一つにする
- 押した後に何が起きるか分かる文言にする
- 相談CTAは相談論点を示す
- 広告表記は読み飛ばされるほど小さくしない
- 公式リンクと広告リンクの見た目・説明を区別する

## アクセシビリティ

- キーボード操作とフォーカス表示を保つ
- 見出し階層とランドマークを正しく使う
- 図解に代替テキストまたは本文説明を付ける
- 折りたたみの開閉状態を支援技術で理解できるようにする
- タップ領域、コントラスト、読み上げ順をモバイルで確認する

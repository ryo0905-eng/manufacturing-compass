# Design System

## デザイン方針

Manufacturing Compass は、単なる読み物サイトではなく、診断結果を見ながら自分のキャリアの現在地を理解するプロダクトです。

UI は「Quiet RPG for engineers」を目指す。

転職・年収・市場価値を扱うため、幼いゲーム表現や煽りは避ける。一方で、ユーザーが自分のキャリアを探索している感覚を持てるように、静かな RPG / クエスト UI の要素を取り入れる。

- 信頼できる
- 直感的
- 少しワクワクする
- 診断結果が読みやすい
- 数値やスコアが分かりやすい
- アフィリエイトサイトっぽく見えない

## 世界観

Manufacturing Compass は、半導体キャリアを探索するための静かなクエスト UI として表現する。

- 診断開始: Start Quest
- 質問: Stage 1 / 4
- 市場価値スコア: Career Power
- 年収レンジ: Reward Range
- 伸ばすべきスキル: Skill Tree
- 今日やること: Today Quest
- 今狙える会社: Available Routes
- 将来狙える会社: Next Area

言葉は大人向けに抑え、過度なファンタジー表現は使わない。

避ける表現:

- 勇者
- ラスボス
- 討伐
- レベルが低い
- ゲームに勝つ / 負ける

使ってよい表現:

- Quest
- Route
- Stage
- Skill
- Power
- Next Area
- Mission

## 画面の主役

主役は診断結果。

以下を視覚的に分かりやすくする。

- 市場価値スコア
- 想定年収レンジ
- 今狙える会社
- 将来狙える会社
- 伸ばすべきスキル
- 今日やること

## 色の方向性

- ベース: 黒に近いネイビー、白、薄いグレー
- テキスト: 黒に近いニュートラル、または白
- メイン: 信頼感のあるシアン
- 成長・可能性: グリーン
- 年収・市場価値: 控えめなゴールド
- 注意・不確実性: 落ち着いたアンバー
- アクセント: レトロゲーム風の小さなドット、細いグリッド、控えめな発光

過度なグラデーション、原色の多用、ファミコン風への振り切りは避ける。

## タイポグラフィ

- 診断結果の数字は大きく見せる
- 本文は短くする
- 長文説明より、カード、ステップ、チェックリストを使う
- スマホで読みやすい余白を保つ

## 診断 UI

- 質問は1画面に詰め込みすぎない
- 1問ずつ進むステップ式にする
- 選択肢はコマンド選択のように見せる
- 進捗は Stage / EXP バーのように見せる
- 結果は「判定」ではなく「現在地」として見せる
- 市場価値スコアは前向きな指標として扱う

## 結果 UI

結果ページはステータス画面風に、以下の順番で表示する。

1. 市場価値スコア
2. 想定年収レンジ
3. 今狙える会社
4. 伸ばすべき要素
5. 今日やること
6. 30日、90日、6か月ロードマップ
7. CTA

## CTA デザイン

- CTA は診断結果と対応させる
- 「相談する」だけではなく、相談論点を明示する
- 英語学習 CTA は、英語が市場価値に効く理由を示した後に置く
- 広告表記は小さくても明確に出す

## 企業ページデザイン

企業ページは診断結果の裏付けとして扱う。

- 会社概要より、転職者に関係する情報を上に出す
- 年収レンジ、職種、英語、必要経験、準備ポイントを見やすくする
- 企業情報は情報ソースを明記する

## モバイルファースト

- 診断はスマホで完了できることを最優先にする
- 選択肢はタップしやすくする
- CTA は押しやすくする
- 長い表は避け、カード型にする

# UI Consistency Audit

監査日: 2026-07-21

## 目的と判定基準

公開ページが [`design-system.md`](./design-system.md) の色、文字階層、情報密度、装飾、例外条件に沿っているかを確認した記録です。UIルールの正本は `design-system.md` であり、この文書は現時点の適用状況と移行順を示します。

`src/app/**/page.tsx` の25ルートテンプレートを対象に、ページ構造、適用されるCSS、CSSの上書き順を確認しました。動的ルートはテンプレート単位で判定しています。共有された業界地図の画面も比較対象にしました。実行環境のブラウザ接続を利用できなかったため、今回の全ルート実表示確認は未実施です。

判定は次の4種類です。

- 準拠: 現在の共通ルールと同種ページの密度に概ね合う
- 承認済み例外: 目的に沿った限定的な例外で、通常ページへ広がっていない
- 一部要調整: 全体は使えるが、一部に旧装飾が残る
- 要移行: ページの中心表現が旧UIで、共通ルールとの差が大きい

## 移行優先順位

### 1. `/companies/[slug]`

37社へ影響するSEO資産であり、閲覧範囲が最も広いため最優先です。見出しを32〜36pxへ戻し、青緑のセクション帯、準備度背景、タグ装飾を中立色へ統合します。企業固有情報、出典、更新日、準備ポイントの階層は維持します。

### 2. `/tools`

子ページのCp・Cpk、DoE、管理図、Gage R&Rは新しい学習UIへ揃っています。入口だけが旧配色のため、中立色、短い説明、ツールの学習目的を中心に再構成します。

### 3. `/career-consultation`

相談前の論点整理という役割は維持し、巨大なマーケティング見出しと金色グラデーションを、通常の読み物・CTAルールへ移行します。

### 4. `/career-agents`

ページ全体ではなく、相談テーマ表と一部状態表示を調整します。広告であることを示す開示表現は意味のある例外として残し、装飾目的のグリッドだけを除きます。

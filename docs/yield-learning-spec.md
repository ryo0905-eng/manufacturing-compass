# 歩留まり・FPY・RTY インタラクティブ学習ツール仕様

最終更新日: 2026-07-20

## 目的

最終的な良品数だけでなく、各工程で「一度で合格したか」「手直しで救済されたか」「廃棄されたか」を分けて考えられるようにする。単工程の歩留まり、First Pass Yield（FPY）、複数工程のRolled Throughput Yield（RTY）を操作し、どの工程を改善すると全体へ効くかを学ぶ。

Excel、MES、歩留まり管理システムを置き換えない。実際の製造データを集計する前に、分母、工程境界、手直しの扱い、複数工程の積み上がりを理解する学習体験とする。

## 想定ユーザー

- 歩留まり、直行率、FPY、RTYの違いを学び直したい製造・品質エンジニア
- 最終良品率だけで工程を評価している初学者
- 手直しや再検査を含む現場で、改善対象を判断したい実務者
- 半導体製造の多工程で、小さなロスが累積する意味を理解したい人

## 最初に答える問い

1. 最終的に良品なら、工程は良好と言えるか
2. 歩留まりとFPYは何が違うか
3. 各工程が98%でも、多工程では全体がどうなるか
4. 手直しは良品を救済する一方で、何を隠すか
5. どの工程から改善するとRTYへ最も効くか
6. 不良品数と欠陥数はなぜ同じとは限らないか

## 用語と計算定義

ページ内では「歩留まり」だけで曖昧にせず、分母と手直しの扱いを併記する。

### 単工程の数量

```text
投入数 = 一発合格数 + 初回不合格数
初回不合格数 = 手直し成功数 + 廃棄数 + 未完了数
手直し後良品数 = 一発合格数 + 手直し成功数
```

初期版では未完了、保留、仕掛かりを扱わず、初回不合格は手直し成功または廃棄へ確定する単純モデルとする。

### FPY（First Pass Yield）

```text
FPY = 一発合格数 ÷ 投入数
```

廃棄、再処理、再検査、オフライン修理へ回らず、一度で要求を満たした割合として扱う。直行率という日本語を併記するが、社内定義が異なる場合があることを明示する。

### 手直し後歩留まり

```text
手直し後歩留まり = 手直し後良品数 ÷ 投入数
```

最終良品率、出荷歩留まりなどの名称は工程境界によって意味が変わるため、初期版では「手直し後歩留まり」と表示する。

### RTY（Rolled Throughput Yield）

直列につながる工程のFPYを掛け合わせる。

```text
RTY = FPY₁ × FPY₂ × … × FPYₙ
```

一つも手直しや再処理を必要とせず、全工程を一度で通過できる割合の学習指標として示す。

### 手直し後の累積歩留まり

```text
累積手直し後歩留まり
  = 工程1の手直し後歩留まり × … × 工程nの手直し後歩留まり
```

RTYと並べることで、最終的な良品は多くても、工程内部で手直しが発生している状態を示す。

## 誤解しやすい数量

### 不良品数と欠陥数

- 不良品数: 基準を満たさない製品・部品の個数
- 欠陥数: 一つの製品に複数存在しうる不具合箇所・不適合項目の数

10個の不良品に合計14件の欠陥がある場合、不良品数は10、欠陥数は14となる。初期版では概念比較までとし、DPU、DPO、DPMO、Poissonモデルは将来拡張とする。

## 初期スコープ

- 1000個投入の理論モデルを初期値とする
- 2〜10工程の直列工程を扱う
- 各工程にFPYと、初回不合格に対する手直し成功率を持たせる
- 工程別FPY、手直し後歩留まり、RTY、累積手直し後歩留まりを即時更新する
- 工程別の一発合格、手直し、廃棄の理論数量を表示する
- 学習データと操作はブラウザ内で完結し、保存・外部送信しない
- 小数による期待数量と、丸めた表示数量を区別する

### 初期版で扱わないもの

- 分岐、合流、並列工程、再入工程
- 複数回の手直しループ
- 仕掛かり、保留、検査待ち、サンプリング検査
- ロット、ウェハ、ダイの階層集計
- 欠陥密度モデル、ウェハマップ、クラスタリング
- DPU、DPO、DPMO、シグマレベル
- 実データ入力、CSV、MES連携、帳票出力
- 金額原価、工数、リードタイムの正式計算

## 学習フロー

### Step 1. 100個を一つの工程へ流す

一発合格、手直し、廃棄へ分かれるフローを表示する。FPYを動かすと一発合格数が変わり、手直し成功率を動かすと手直し後良品と廃棄が変わる。

最初に「良品数」ではなく、どの経路で良品になったかを見る。

### Step 2. 同じ最終歩留まりでも中身が違う

次の二つを並べる。

- 一度で合格する工程
- 初回不合格が多いが、手直しで救済する工程

手直し後歩留まりが同程度でも、FPY、再投入、検査、滞留の負担が異なることを体感する。初期版では正式な原価・時間を計算せず、「追加処理数」として示す。

### Step 3. 工程を増やしてRTYを見る

各工程のFPYが98%の状態で、工程数を2〜10へ増やす。単工程では小さく見える2%の初回不合格が、多工程のRTYでは累積することを示す。

```text
98% × 98% × … × 98%
```

数式だけでなく、1000個のうち全工程を一度で通過する期待数量として表示する。

### Step 4. 手直しで隠れるロスを見る

RTYと累積手直し後歩留まりを同じグラフへ表示する。二つの差を「手直しで救済された割合」として見せ、最終良品率だけでは工程内のロスが見えにくいことを説明する。

### Step 5. 改善する工程を選ぶ

工程カードを選び、FPYを1ポイント改善した場合のRTY変化を比較する。最もFPYが低い工程、処理数が多い工程、改善可能幅が大きい工程を区別する。

改善候補は自動決定せず、次も確認候補として示す。

- 不良モードと件数
- 製品、設備、材料、作業者、時間帯による層別
- 測定システム
- 工程の安定性
- 手直し工数、滞留、再検査
- 改善費用と製品リスク

### Step 6. 欠陥と不良品を分ける

同じ不良品数の中へ欠陥を追加し、欠陥数だけが増える操作を用意する。「不良品率」と「欠陥数ベースの指標」は別の問いに答えることを体感する。

## 学習シナリオ

初期版では次のプリセットを用意する。

1. 高FPYの少数工程
2. 各工程98%だが工程数が多い
3. 最終歩留まりは高いが手直しが多い
4. 一つのボトルネック工程がある
5. 複数工程に小さなロスが分散している

プリセット後も工程数、工程別FPY、手直し成功率を変更できる。

## UI構成

```text
学習ステップ
  01 一発合格と手直しを分ける
  02 最終歩留まりの中身を見る
  03 多工程のRTYを見る
  04 改善する工程を選ぶ

工程フロー
  → 工程別設定
  → FPY / 手直し後歩留まり
  → RTY / 累積手直し後歩留まり
  → 数量への換算
  → 次に確認するデータ
```

### 初期画面

- 最初は単工程の100個モデルで、一発合格、手直し、廃棄を見せる
- 用語の前に物の流れを矢印と数量で示す
- FPYと手直し後歩留まりは色だけでなくラベルと線種を分ける
- 指標カードには必ず分子・分母を表示できる説明を置く
- 詳細な計算式、前提、関連指標は折りたたむ

### 多工程表示

- PCでは工程を左から右へ並べる
- スマートフォンでは横スクロールまたは縦フローにする
- 工程数を増やしても文字を縮小しない
- 工程ごとの投入、一発合格、手直し、廃棄を必要に応じて展開する
- 最も低い工程だけを赤く断定せず、改善影響と確認候補を文章で示す

### グラフ

- Seabornに近い薄いグレー背景と落ち着いた青を基本にする
- FPY・RTYは青、手直し後歩留まりは緑、手直し・廃棄は橙・赤を限定的に使う
- 工程別FPYの棒グラフと、工程を進む累積RTYの線グラフを優先する
- パーセントだけでなく、1000個換算の数量を併記する

## 計算・丸め方針

- 内部計算は0〜1の小数で保持する
- RTYは丸め前の工程別FPYを掛ける
- 表示は原則としてパーセント小数1桁、数量は期待値として小数1桁または整数に丸める
- 途中表示で丸めた値を次工程の計算へ再利用しない
- FPY、手直し率、廃棄率の範囲を検証し、負の数量や100%超を許可しない
- 初回不合格に対する手直し成功率と、投入全体に対する手直し率を混同しない

## 半導体製造への適用範囲

半導体は多工程、再入工程、ウェハ・ダイ階層、検査ビン、スクラップ、保留などがあるため、実際の歩留まり管理は単純な直列積より複雑である。

初期版では工程名プリセットとして、成膜、パターニング、加工、検査などの一般化した名称を使えるようにする。ただし実在プロセスの代表歩留まりや標準値とは表現せず、直列工程を理解する学習例と明記する。

将来、半導体固有の体験を追加する場合は、次を別モジュールにする。

- ウェハ投入から良品ダイまでの階層
- ウェハマップと空間的欠陥
- 欠陥密度とダイ面積
- ウェハソート、組立、最終検査の段階歩留まり
- ビン分類と救済可能品

## 誤解防止

- 最終良品率が高くても、FPYが高いとは限らない
- RTYが低いことだけで、原因工程を断定できない
- 検査で不良を見つけても、不良の発生原因を減らしたことにはならない
- 手直し成功品も、追加工数、再検査、滞留、品質リスクを伴いうる
- 工程境界と分母が違う歩留まりを直接比較しない
- 工程が統計的管理状態か、測定値を信頼できるかは別に確認する
- 学習用の独立・直列モデルを、実工程の生産予測へ直接使用しない

## 実装構成

- ルート候補: `/tools/yield`
- ページと静的解説: Server Component
- 工程追加、スライダー、フロー、グラフ: 専用Client Component
- 計算ロジックと型: `src/lib/yield/`
- プリセットと説明文: UIから分離した静的データ
- SVGでグラフを実装し、新しい重量級チャートライブラリを追加しない
- 外部API、データベース、ユーザー保存を使用しない

## 既存ツールとの接続

```text
Gage R&R: 測定値を信頼できるか
  → 管理図: 工程が安定しているか
  → 歩留まり: どこで良品を失っているか
  → パレート・層別: 何の不良が多いか
  → DoE: 条件をどう改善するか
  → Cp・Cpk: 規格に対する能力を確認する
```

- Gage R&Rと管理図から歩留まりへ内部リンクする
- 歩留まりの改善候補から将来のパレート・層別へつなぐ
- 工程条件を改善する文脈ではDoE、規格能力ではCp・Cpkへつなぐ
- 転職CTAではなく、次の技術判断を主導線にする

## 参考定義と出典方針

- FPYの中心定義はASQ Quality Glossaryを参照する
- RTYは工程別FPYの積として扱うが、欠陥数を使う定義や複雑な経路では解釈が異なることを明示する
- 実装時は計算定義をページ内に表示し、出典URLと確認日を保持する
- 半導体固有の数値や標準値は推測で追加しない

参考:

- ASQ, Quality Glossary: https://asq.org/quality-resources/quality-glossary
- ASQ, Six Sigma Rolled Throughput Yield: https://asq.org/quality-resources/articles/six-sigma-rolled-throughput-yield?id=2be8d7ffb4eb4ccda87c777ffac4e478

## 初期版の完了条件

- 一発合格、手直し成功、廃棄の流れを操作できる
- FPYと手直し後歩留まりの違いを数量で説明できる
- 2〜10工程を切り替え、RTYの累積を確認できる
- 5つのシナリオを切り替えられる
- 工程別FPYと累積RTYが同時更新される
- 1000個換算の一発通過数と手直し後良品数を確認できる
- 改善する工程を選び、RTYへの影響を比較できる
- 不良品数と欠陥数の違いを操作で確認できる
- 計算ロジックをUIから分離して検証できる
- PCとスマートフォンで主要操作ができる
- `/tools`とトップページからアクセスできる

# Conversion Architecture

## 問題意識

現在の Manufacturing Compass は、半導体業界の情報を整理するサイトとしては成立しているものの、ユーザーが自分の状況を入力し、結果を受け取り、次の行動へ進む「アプリ的な体験」が弱い。

このままだと、薄い企業情報を読んで終わるサイトになりやすく、転職エージェント相談へのコンバージョンが起きにくい。

## 新しい中心体験

中心に置くべき体験は `Career Compass`。

ユーザーは以下を選ぶ。

- いま一番近い経験
- 経験年数
- 英語の状態
- 転職の狙い

その結果として以下を返す。

- 今近い半導体領域
- 今狙いやすい会社
- 将来のストレッチ目標
- 今日からできること
- 半年で近づく準備
- 転職エージェントに相談すべき論点

## 新しい導線

1. トップページ
2. Career Compass
3. 結果を見る
4. 企業詳細・比較を見る
5. 相談すべき論点が明確になる
6. 転職エージェント CTA をクリックする

## ページの役割

- `/`: 世界観と Career Compass への入口
- `/career-compass`: メインのアプリ体験
- `/industry-map`: 業界の理解を補助するページ
- `/companies`: 企業を探すページ
- `/companies/[slug]`: 企業研究と準備ポイント
- `/compare`: 迷っている企業を比較するページ

## マネタイズ上の考え方

転職エージェント CTA は、単に「相談する」ではなく、ユーザーの結果に合わせて相談論点を作ってから出す。

例:

- 品質保証経験を半導体品質職・プロセス職へどう接続できるか相談する
- 生産技術経験で狙えるプロセス職と、半年後に広がる選択肢を相談する
- 設備保全経験を装置メーカー・半導体工場のどちらへ寄せるべきか相談する

CTA は広告ではなく、ユーザーの不確実性を減らす次の行動として置く。

## MVP の範囲

Phase 1.5 では、ログイン、AI、求人連携、個人情報保存は行わない。

静的ルールベースの診断として実装し、需要検証を優先する。

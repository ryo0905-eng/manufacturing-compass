# Conversion Architecture

## 問題意識

企業一覧や業界地図だけでは、ユーザーは自分ごと化しにくい。

転職エージェントのアフィリエイトで収益化するには、ユーザーが「自分は今どのくらいの市場価値があり、何をすれば伸びるのか」を理解した状態を作る必要がある。

## 中心体験

中心体験は、半導体キャリア市場価値診断。

ユーザーは以下を知りたい。

- 自分の市場価値
- 想定年収レンジ
- 今狙える会社
- 将来狙える会社
- 年収を伸ばすために必要なこと
- 今日やること

## 新しいファネル

```text
SEO / SNS / 直接流入
↓
トップページ
↓
半導体キャリア市場価値診断
↓
診断結果
↓
企業詳細 / 比較 / 学習コンテンツ
↓
転職エージェント CTA
英語学習 CTA
資格・学習 CTA
```

## ページの役割

- `/`: 診断開始を主役にする
- `/diagnosis`: 市場価値診断
- `/diagnosis/result`: 市場価値、年収レンジ、次の行動
- `/companies`: 診断結果に出る企業の根拠
- `/companies/[slug]`: 企業別の仕事内容、年収目安、準備ポイント
- `/compare`: 迷っている企業を比較
- `/industry-map`: 業界理解の補助
- `/agents`: 転職エージェント比較
- `/english`: 英語学習導線

## CTA 設計

CTA は診断結果に応じて出し分ける。

### 転職エージェント

表示タイミング:

- 今狙える会社が出た後
- 想定年収レンジが出た後
- 職務経歴書の見せ方が課題になった時

### 英語学習

表示タイミング:

- 外資系・グローバル職がストレッチ目標に出た時
- 英語力が市場価値アップ要因として出た時

### 資格・学習

表示タイミング:

- QC、統計、データ分析、半導体プロセスが不足要素に出た時

## CV に必要な心理状態

ユーザーが CTA をクリックする前に必要な状態:

1. 自分の現在地が分かった
2. 今のまま狙える会社が見えた
3. もっと伸びる余地が見えた
4. 何をすればいいか分かった
5. でも自分だけでは求人選びや職務経歴書の見せ方に不安がある

この状態を作ってから CTA を出す。

## Phase 1.5 MVP

- 静的ルールベース診断
- 年収レンジは目安表示
- 企業 DB は根拠ページ

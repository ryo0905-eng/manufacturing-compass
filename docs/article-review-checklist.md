# Article Review Checklist

## 実体験

- [ ] 実際に経験した内容だけが「実体験」として書かれている
- [ ] 自分の担当範囲とチームの成果が分かれている
- [ ] 失敗や不確実性が都合よく削除されていない
- [ ] 体験をすべての読者に当てはまる一般論にしていない

## 数字

- [ ] 改善率、効果額、年収に根拠がある
- [ ] 対象期間と対象範囲が分かる
- [ ] 実測、試算、概算が区別されている
- [ ] 因果関係を断定しすぎていない

## 機密情報

- [ ] 勤務先、顧客、製品を意図せず特定できない
- [ ] 非公開の工程条件、設備設定、材料、社内資料が含まれていない
- [ ] 転職先や選考に関する非公開情報が含まれていない
- [ ] 公開してよい数字だけを使用している

## 公開情報

- [ ] 企業情報、求人条件、業界情報に一次情報のURLがある
- [ ] 出典の内容と記事の主張が一致している
- [ ] 確認日が記録されている
- [ ] 古い情報や変動する情報に注意書きがある

## 読者への価値

- [ ] 読者の悩みが一つに絞られている
- [ ] 冒頭で結論が分かる
- [ ] 読者が今日できる行動がある
- [ ] 不安を過度に煽っていない
- [ ] 今すぐ狙えることと将来の準備が分かれている

## 読みやすさと本人らしさ

- [ ] 8分以上の記事に、内容固有の情報ブロックが3つ以上ある
- [ ] 本文だけの段落が3つ以上連続していない
- [ ] スマホで1段落が長くなりすぎていない
- [ ] 求人要件、比較、手順は文章ではなく表やカードへ整理されている
- [ ] RYO本人の率直な言葉が最低1か所残っている
- [ ] 一般論、注意書き、結論を重複して書いていない
- [ ] AI画像や内容と関係の薄いストック画像で余白を埋めていない
- [ ] PC・スマホの全ページ表示で視線誘導を確認した

## 公開設定

- [ ] タイトルと description が本文を正確に表している
- [ ] 公開日、更新日、著者、確認者が設定されている
- [ ] 関連ガイド、企業、診断への内部リンクがある
- [ ] 構造化データへ公開日と更新日が入っている
- [ ] typecheck、lint、E2E、build が成功している

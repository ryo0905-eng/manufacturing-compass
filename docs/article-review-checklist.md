# Article Review Checklist

EXP は全項目、REF は「公開情報」「読者への価値」「公開設定」と該当する数値・機密・読みやすさの項目を確認します。該当しない項目を満たすために実体験や図表を追加しません。

## 実体験（EXP）

- [ ] 実際に経験した内容だけが「実体験」として書かれている
- [ ] 自分の担当範囲とチームの成果が分かれている
- [ ] 失敗や不確実性が都合よく削除されていない
- [ ] 体験をすべての読者に当てはまる一般論にしていない

## 数字

- [ ] 改善率、効果額、年収に根拠がある
- [ ] 対象期間と対象範囲が分かる
- [ ] 実測、試算、概算が区別されている
- [ ] 因果関係を断定しすぎていない

## 機密情報

- [ ] 勤務先、顧客、製品を意図せず特定できない
- [ ] 非公開の工程条件、設備設定、材料、社内資料が含まれていない
- [ ] 転職先や選考に関する非公開情報が含まれていない
- [ ] 公開してよい数字だけを使用している

## 公開情報

- [ ] 企業情報、求人条件、業界情報に一次情報のURLがある
- [ ] 出典の内容と記事の主張が一致している
- [ ] 確認日が記録されている
- [ ] 古い情報や変動する情報に注意書きがある

## 読者への価値

- [ ] 読者の悩みが一つに絞られている
- [ ] 冒頭で結論が分かる
- [ ] 読者が今日できる行動がある
- [ ] 不安を過度に煽っていない
- [ ] 今すぐ狙えることと将来の準備が分かれている

## 読みやすさと本人らしさ

- [ ] 内容の関係を理解しやすくする箇所だけに図表・情報ブロックを使っている
- [ ] 本文と図表・カードで同じ説明を重複していない
- [ ] スマホで1段落が長くなりすぎていない
- [ ] 求人要件、比較、手順は、対応関係が明確になる場合に表やカードへ整理されている
- [ ] EXP記事では、RYO本人の確認済みの率直な言葉が残っている
- [ ] 一般論、注意書き、結論を重複して書いていない
- [ ] AI画像や内容と関係の薄いストック画像で余白を埋めていない
- [ ] 画面確認を行う場合、PC・スマホで読みやすさと視線誘導を確認した

## 公開設定

- [ ] タイトルと description が本文を正確に表している
- [ ] 公開日、更新日、著者、確認者が設定されている
- [ ] 関連ガイド、企業、Career Compass など、検索意図に合う内部リンクがある
- [ ] 構造化データへ公開日と更新日が入っている
- [ ] `AGENTS.md` に従い、変更内容に必要な検証と差分確認が完了している
